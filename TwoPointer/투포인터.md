# 투포인터

1차원 배열에서 두 개의 포인터를 조작하여 원하는 결과를 얻는 알고리즘입니다.

## 원리

투포인터의 원리는 백준 2303번 문제인 수들의 합2를 예시로 들어 설명합니다. 


> ## 문제
> N개의 수로 된 수열 A[1], A[2], …, A[N] 이 있다. 이 수열의 i번째 수부터 | j번째 수까지의 합 A[i] + A[i+1] + … + A[j-1] + A[j]가 M이 되는 경우의 수를 구하는 프로그램을 작성하시오.
> 
> ## 입력
> 첫째 줄에 N(1 ≤ N ≤ 10,000), M(1 ≤ M ≤ 300,000,000)이 주어진다. 다음 줄에는 A[1], A[2], …, A[N]이 공백으로 분리되어 주어진다. 각각의 A[x]는 30,000을 넘지 않는 자연수이다.

투포인터에서는 start, end라는 두개의 포인터를 사용합니다. start는 배열의 앞쪽을 가리키는 인덱스이고, end는 배열의 뒤쪽을 가리키는 인덱스입니다.

여기서 원하는 부분 배열의 합을 구해야 합니다.

- 부분배열의 합 < 구해야 하는 값
    
    end를 오른쪽으로 한 칸이동하여 부분배열의 합을 증가

- 부분배열의 합 > 구해야 하는 값hk

    start를 오른쪽으로 한 칸 이동하여 부분배열의 합을 감소

## 시간 복잡도

투포인터는 두 포인터가 배열의 마지막에 도달할 경우 종료되므로 최악의 경우에도 $O(2n)$이 됩니다.
